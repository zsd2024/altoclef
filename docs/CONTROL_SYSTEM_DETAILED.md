# AltoClef 控制系统详细文档

## 概述

AltoClef的控制系统是整个自动化机器人的核心组件，负责处理玩家的各种操作，包括物品栏管理、输入控制、战斗控制等。该系统通过多个专门的控制器类协同工作，实现对Minecraft游戏中玩家行为的全面控制。

## 系统架构

控制系统主要由以下几个核心组件构成：

### 1. InputControls (输入控制器)

输入控制器负责管理Minecraft游戏中的各种输入操作，包括移动、攻击、跳跃、潜行等。它提供了一套高级API来处理按键状态和输入事件。

#### 核心功能：
- **按键状态管理**：跟踪和控制按键的按下、释放状态
- **单帧输入**：提供单帧按下输入的功能，避免持续按下
- **视角控制**：强制设置玩家的视角
- **输入同步**：确保输入操作被游戏正确识别

#### 主要方法：
- `tryPress(Input input)`：尝试按下输入（仅一帧）
- `hold(Input input)`：持续按下输入
- `release(Input input)`：释放输入
- `isHeldDown(Input input)`：检查输入是否被按下
- `forceLook(float yaw, float pitch)`：强制设置视角

#### 实现特点：
- 使用队列管理需要释放的输入
- 使用集合跟踪等待释放的点击操作
- 通过`KeyBinding.onKeyPressed`确保输入被游戏正确注册

### 2. SlotHandler (物品栏处理器)

物品栏处理器管理玩家的物品栏操作，包括物品移动、装备、交易等。它是与Minecraft物品栏交互的核心组件。

#### 核心功能：
- **物品栏操作**：处理各种物品栏点击操作
- **装备管理**：自动装备和卸下物品
- **计时控制**：控制物品栏操作的速度，避免过快操作
- **物品交换**：在不同槽位间移动和交换物品

#### 主要方法：
- `clickSlot(Slot slot, int mouseButton, SlotActionType type)`：点击槽位
- `forceEquipItem(Item toEquip)`：强制装备物品
- `forceDeequipHitTool()`：强制卸下攻击工具
- `forceEquipItemToOffhand(Item toEquip)`：强制将物品装备到副手
- `canDoSlotAction()`：检查是否可以执行物品栏操作

#### 实现特点：
- 使用计时器控制操作频率，防止过快操作
- 支持强制操作以绕过计时器限制
- 提供多种装备和卸下策略

### 3. PlayerExtraController (玩家额外控制器)

玩家额外控制器处理除基本移动外的其他玩家操作，如攻击实体、挖掘方块等。

#### 核心功能：
- **实体攻击**：向实体发起攻击
- **范围检查**：检查实体是否在攻击范围内
- **挖掘追踪**：跟踪当前挖掘的方块状态
- **事件监听**：监听挖掘开始和取消事件

#### 主要方法：
- `attack(Entity entity)`：攻击实体
- `inRange(Entity entity)`：检查实体是否在范围内
- `getBreakingBlockPos()`：获取正在挖掘的方块位置
- `isBreakingBlock()`：检查是否正在挖掘方块
- `getBreakingBlockProgress()`：获取挖掘进度

#### 实现特点：
- 通过事件总线监听挖掘事件
- 自动计算攻击范围
- 与Minecraft实体系统直接交互

### 4. KillAura (击杀光环控制器)

击杀光环是AltoClef的战斗控制系统，自动攻击附近的敌对实体，提供自动战斗功能。

#### 核心功能：
- **目标管理**：跟踪和管理待攻击目标
- **智能攻击**：根据策略自动攻击实体
- **盾牌保护**：在需要时启用盾牌保护
- **武器选择**：自动选择最佳武器

#### 主要方法：
- `applyAura(Entity entity)`：将实体添加到攻击目标
- `setRange(double range)`：设置攻击范围
- `tickStart()`：每tick开始时的初始化
- `tickEnd(AltoClef mod)`：每tick结束时的处理
- `startShielding(AltoClef mod)`：开始盾牌保护
- `stopShielding(AltoClef mod)`：停止盾牌保护

#### 攻击策略：
- **OFF**：关闭攻击
- **FASTEST**：最快攻击（立即攻击所有目标）
- **DELAY**：延迟攻击（等待攻击冷却）
- **SMART**：智能攻击（结合多种条件判断）

#### 实现特点：
- 支持多种攻击策略
- 自动管理武器装备
- 智能盾牌保护机制
- 考虑玩家状态（进食、掉落等）

## 系统交互

控制系统各组件之间通过AltoClef主模块协调工作：

1. **SlotHandler** 与 **PlayerExtraController** 协同处理物品和攻击操作
2. **InputControls** 为其他组件提供输入支持
3. **KillAura** 使用其他组件的功能实现自动战斗
4. 各组件共享状态信息，确保操作的一致性

## 线程安全与同步

控制系统的实现考虑了Minecraft的单线程特性，所有操作都在游戏主线程中执行，通过计时器和状态管理确保操作的正确时序。

## 性能优化

1. **操作节流**：通过计时器限制物品栏操作频率
2. **智能缓存**：缓存常用物品和状态信息
3. **事件驱动**：使用事件总线减少不必要的轮询
4. **条件检查**：在执行操作前进行必要条件检查

## 总结

AltoClef的控制系统是一个高度模块化和协调的架构，通过各个专门的控制器实现对Minecraft游戏中玩家行为的全面自动化控制。这种设计既保证了功能的完整性，又确保了系统的可维护性和扩展性。
